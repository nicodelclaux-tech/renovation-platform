# Prompting Rules

## Rule 1 — Preserve fixed architecture

Never alter fixed architecture unless explicitly instructed.

## Rule 2 — Use approved references

When a user approves a visual direction, use it as the baseline for future iterations.

## Rule 3 — Separate design intent from edit instruction

Design intent is stable. Edit instruction is local to the current generation.

## Rule 4 — Store every prompt

Every generation must save the exact prompt, source images, model, and parent generation.
